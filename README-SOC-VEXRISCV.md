# SoC LiteX+VexRiscv - TensorFlow Lite Integration

Laboratório de Desenvolvimento de Software (LDS) - IFCE  
Integração completa do SoC RISC-V LiteX+VexRiscv com TensorFlow Lite Micro.

## Visão Geral

O SoC LiteX+VexRiscv agora está oficialmente integrado aos scripts existentes de TensorFlow Lite, permitindo geração automática de código para esta plataforma RISC-V customizada.

### Características do SoC

- **CPU**: VexRiscv RV32IM @ 50MHz
- **FPGA**: ColorLight i5 (ECP5 LFE5U-25F)
- **Memória**: 64KB ROM + 64KB SRAM (otimizada)
- **Framework**: LiteX + VexRiscv
- **Comunicação**: UART @ 115200 bps
- **Toolchain**: Docker `carlosdelfino/colorlight-risc-v:latest`

## Integração com Scripts Existentes

### TensorFlow_GUI_Simple.py

O SoC LiteX+VexRiscv foi adicionado ao dropdown de microcontroladores:

```
MCU: [ESP32 (Arduino), ESP32 (ESP-IDF), STM32, Arduino Nano, SoC LiteX+VexRiscv]
```

**Funcionalidades disponíveis:**
- Geração automática de código C
- Suporte a quantização Int8 e Float32
- Makefile específico para LiteX
- README com instruções completas
- Integração com workflow existente

### Arquivos Gerados

Quando selecionado "SoC LiteX+VexRiscv", o sistema gera:

1. **Modelo TensorFlow Lite** (.tflite)
2. **Header C** (.h) com array de bytes
3. **Firmware C** (soc_tensorflow_model.c)
4. **Makefile** para compilação LiteX
5. **README.md** com instruções detalhadas

## Como Usar

### Método 1: Interface Gráfica

```bash
source venv/bin/activate
cd scripts
python3 TensorFlow_GUI_Simple.py
```

**Passos:**
1. Treine seu modelo normalmente
2. Clique em "Exportar Modelo"
3. Selecione "SoC LiteX+VexRiscv" no dropdown MCU
4. Escolha quantização "Int8 (8-bit)" (recomendado)
5. Clique em "Exportar"

### Método 2: Workflow Automatizado

```bash
./gerar_modelo_soc.sh nome_do_modelo
```

## Performance e Memória

### Configuração Otimizada

```

  ROM (64KB): Modelo + Firmware
    Modelo TensorFlow: ~19KB (29.7%)

  SRAM (64KB): Runtime + Buffers
```

## Compilação e Gravação

### 1. Gerar SoC com Modelo

```bash
docker run --rm -v $(pwd):/workspace -w /workspace \
    carlosdelfino/colorlight-risc-v:latest \
    python3 soc.py --board i5 --sys-clk-freq 50e6 \
    --build --output-dir build-soc-tflite
```

### 2. Compilar Firmware

```bash
make -C modelo_soc BUILD_DIR=../build-soc-tflite
```

### 3. Gravar na FPGA

```bash
./flash-tensorflow.sh i5 build-soc-tflite
```

### 4. Testar via UART

```bash
./test-tensorflow.sh /dev/ttyACM0 115200
```

## Troubleshooting

### Problemas Comuns

**Modelo muito grande:**
```
ERRO: Modelo excede memória ROM (64KB)
Solução: Reduza camadas ocultas ou use pruning
```

**Compilação falha:**
```
ERRO: model_int8.h não encontrado
Solução: Verifique se exportação incluiu header
```

**Inferência lenta:**
```
AVISO: Inferência > 500µs
Solução: Use quantização Int8 ou simplifique modelo
```

### Debug via Serial

```bash
# Monitoramento completo
minicom -D /dev/ttyACM0 -b 115200

# Comandos de debug
echo "info" > /dev/ttyACM0
echo "test" > /dev/ttyACM0
```
